How to avoid Github/merge conflicts with Rmd-files

Posted on March 6, 2017

One nice features of .rmd files is that version control systems, such as git and github, can (quite) easily be combined. However, in my experience, merge conflicts are not so uncommon. That raises the question how to avoid merge conflicts when syncing with Github? Here’s a quick overview on what to do to that hassle: Sync often. Hard wrap the lines to approx. 80 characters.
